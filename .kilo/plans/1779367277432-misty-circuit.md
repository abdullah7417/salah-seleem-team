# Plan: Update TransformationVideos to Show API Videos

## Context

The `TransformationVideos.jsx` component currently uses `highlighted_transformations` with YouTube URLs. The API now returns video items with a different structure (Vimeo/direct MP4 URLs, thumbnails, and image fallbacks). We need to update the component to use this new API data.

### API Video Item Structure
```json
{
  "id": 26,
  "title": "Shoaib Abdelhafez",
  "description": "...",
  "image_url": "https://...Image-BG.png",
  "thumbnail": "https://i.vimeocdn.com/video/...",
  "url": "https://player.vimeo.com/.../file.mp4?...",
  "vimeo_url": "/videos/1054162417",
  "ready": 1,
  "ordering": 16
}
```

## Changes Required

### File: `merged/src/landings/lifestyle/sections/TransformationVideos.jsx`

1. **Update data source**: Replace `highlighted_transformations` with `apiData.videos` (then fallback to `staticThumbs` if empty).

2. **Update `videoThumbs` memo** to map API video fields:
   - `id` → `v.id`
   - `hasUrl` → `!!v.url` (whether video URL exists)
   - `thumbnail` → `v.thumbnail` (Vimeo CDN thumbnail, used when URL exists)
   - `image_url` → `v.image_url` (fallback image when no URL)
   - `url` → `v.url` (direct MP4 video URL)
   - `title` → `v.title || v.description`

3. **Update SwiperSlide rendering**:
   - **When `url` exists**: Show `thumbnail` as the image, display the play button overlay, wrap in `<a>` tag with `data-fancybox` to open the video on click.
   - **When `url` is null/empty**: Show `image_url` as the image, **hide the play button**, render as a `<div>` instead of `<a>` (no click action).

4. **Update Fancybox config**: The API returns direct MP4 URLs, not YouTube. Update the fancybox options to support HTML5 video playback:
   - Remove `Youtube: { autoplay: 1 }`
   - Fancybox auto-detects MP4 URLs and uses HTML5 video player

5. **Remove YouTube-specific helpers**: Remove `getYouTubeId` function since we no longer need YouTube thumbnail generation. The API provides `thumbnail` directly.

6. **Update static fallback**: Keep `staticThumbs` as fallback when API has no videos, adapting the structure to match the new field expectations.
