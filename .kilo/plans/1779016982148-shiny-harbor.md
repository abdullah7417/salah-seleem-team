# Plan: Replace Static Data with Dynamic API Data

## Goal
Fetch landing page data from the API endpoint `http://salah-seleem-team.test/api/{theme}` based on the current route, replacing static data imports. Fall back to static data when API returns null/empty.

## API Response Structure
The API returns:
```
{ status, message, data: { general_data, membership_plans, highlighted_transformations, benefits, steps, comparisons, programs, plans, memberships, partners, asks, faqs, biologies, biology_steps, videos, ... } }
```

## Architecture: React Context + Custom Hook

---

## Step 1: Update `.env`
- Change `VITE_API_BASE_URL` from `http://salah-seleem-team.test/api/endpoint` to `http://salah-seleem-team.test/api/`

## Step 2: Update `config/landings.js`
- Add `apiEndpoint` field to each config:
  - `"bio-medical"` → `apiEndpoint: "biomedical"`
  - `"lifestyle"` → `apiEndpoint: "lifestyle"`

## Step 3: Create `src/context/LandingDataContext.jsx`
- React context that holds the full API `data` object
- Exports `LandingDataProvider` component and `useLandingData()` hook
- `useLandingData()` returns the full `data` object from API response
- Empty object `{}` as default when no data loaded

## Step 4: Update `src/api/client.js`
- Remove `/endpoint` from default baseURL (it's already handled by `.env`)
- No structural changes needed — `client.get("biomedical")` will call `{baseURL}biomedical`

## Step 5: Create `src/hooks/useLandingApi.js`
- Custom hook: `useLandingApi(apiEndpoint, apiKey)`
- Fetches `GET {apiEndpoint}` using `createClient(apiKey)`
- Returns `{ data, loading, error }`
- Re-fetches when `apiEndpoint` changes (route change)

## Step 6: Update `src/LandingRenderer.jsx`
- Import and call `useLandingApi(config.apiEndpoint, config.apiKey)`
- Wrap rendered sections in `<LandingDataProvider value={apiData}>`
- Pass data down via context instead of props
- Show loading skeleton while fetching, fall back to static data on error

## Step 7: Update Section Components to Use API Data

### Data Mapping (API → Section props):

**Biomedical sections:**

| Section | Static source | API source | Fallback |
|---|---|---|---|
| Hero | `heroReviews` | `data.general_data` for titles + `data.highlighted_transformations` (filtered `show_hero=1`) | Static `heroReviews` |
| BeforeAfter | `beforeAfterCards` | `data.highlighted_transformations` → `{ result: final_result/target, name: user_name, age: derived, duration: ..., condition: type }` | Static `beforeAfterCards` |
| WhoFor | `whoForItems` | `data.benefits` → extract `title + description` | Static `whoForItems` |
| VideoTestimonials | `videoTestimonials` | `data.videos` or `data.videosWithoutChunk` | Static `videoTestimonials` |
| HowItWorks | `howItWorksSteps`, `teamMembers` | `data.steps` for steps, `data.programs` for team | Static arrays |
| Pricing | `pricingOptions`, `pricingFeatures`, `medicalTags` | `data.membership_plans[0].features` for features, `data.memberships` for pricing options | Static arrays |
| FAQ | `faqs` | `data.faqs` → `{ q: question, a: answer }` | Static `faqs` |

**Lifestyle sections:**

| Section | Static source | API source | Fallback |
|---|---|---|---|
| Transformations | `transformations` | `data.highlighted_transformations` → `{ before: image_before, after: image_after, name: user_name, result: target/final_result, ... }` | Static `transformations` |
| TransformationVideos | `videoThumbs` | `data.highlighted_transformations` → `{ image: image_before or image_after }` | Static `videoThumbs` |
| Pricing | `pricingPlans`, `pricingFeatures` | `data.memberships` → `{ id, label: title, price: price, before: price_before, duration: months }`, `data.membership_plans[0].features` → features | Static arrays |
| Partners | `partners` | `data.partners` → extract media URLs | Static `partners` |
| FAQ | `faqs` | `data.faqs` → `{ q: question, a: answer }` | Static `faqs` |
| WhyJoin | `problems`, `benefits` | `data.comparisons` for benefits, `data.asks` for problems | Static arrays |

### Image Fallback Strategy
- For each image field from API (e.g., `image_before`, `image_after`, `image`):
  - If API value is truthy (non-null, non-empty string) → use API URL
  - If API value is null/empty → fall back to static imported image

### How each section will be updated:
1. Import `useLandingData()` from context
2. Import static data as before (for fallback)
3. In component body, destructure API data and map to existing shape
4. Use mapped data with fallback: `apiData || staticData`
5. For images: `apiImage || staticImage`
