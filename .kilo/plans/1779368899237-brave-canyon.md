# Plan: BeforeAfter.tsx — Show API Images on Cards

## Problem
The `CardSlide` component in `BeforeAfter.tsx` always renders `<ImagePlaceholder>` for before/after images, even when the API provides `image_before` / `image_after` URLs. The data mapping in the parent already extracts these into `imageBefore` / `imageAfter` fields, but the card never uses them.

## Reference
The lifestyle `Transformations.jsx` renders actual `<img>` tags with `t.before` / `t.after` src props, falling back to static assets when API images are missing.

## Changes (single file: `BeforeAfter.tsx`)

### 1. Update `CardSlide` to render real images when available

Replace the two `<ImagePlaceholder>` elements inside the grid with conditional rendering:

```
{c.imageBefore ? (
  <div className="relative aspect-[3/4] overflow-hidden bg-[#F0F0EE]">
    <img src={c.imageBefore} alt={`قبل ${c.name}`} className="w-full h-full object-cover" loading="lazy" />
    <span className="absolute top-1.5 right-1.5 rounded px-1.5 py-0.5 text-[9px] bold bg-black/70 text-white">قبل</span>
  </div>
) : (
  <ImagePlaceholder label="قبل" />
)}

{c.imageAfter ? (
  <div className="relative aspect-[3/4] overflow-hidden bg-[#F0F0EE]">
    <img src={c.imageAfter} alt={`بعد ${c.name}`} className="w-full h-full object-cover" loading="lazy" />
    <span className="absolute top-1.5 left-1.5 rounded px-1.5 py-0.5 text-[9px] bold bg-[#47706C] text-white">بعد</span>
  </div>
) : (
  <ImagePlaceholder label="بعد" />
)}
```

This keeps the placeholder as fallback when no image URL exists (static data has no images).

### 2. Add a thin white divider between images (matching lifestyle pattern)

Add a vertical divider `<div>` inside the grid, between the two columns:
```
<div className="absolute inset-y-0 left-1/2 -translate-x-1/2 w-0.5 bg-white/70 z-10" />
```

Add `relative` to the parent grid div.

### 3. No changes to data mapping or Swiper logic
The existing `useMemo` in `BeforeAfter` already correctly maps `t.image_before` → `imageBefore` and `t.image_after` → `imageAfter` from `apiData.transformationHeros`. No changes needed there.

## Summary
- **File changed**: `merged/src/landings/biomedical/sections/BeforeAfter.tsx`
- **Scope**: Only the `CardSlide` component's JSX (lines ~50-67)
- **Behavior**: When API provides image URLs → render `<img>` tags. When not (static fallback) → keep placeholder. Matches the lifestyle Transformations pattern.
