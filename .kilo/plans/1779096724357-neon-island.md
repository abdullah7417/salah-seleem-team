# Plan: Fancybox Video Integration for Transformation Slider

- `TransformationVideos.jsx` has a working Swiper slider with cards (images + play button)
- Cards are `<button>` elements — no video link, no Fancybox binding
- `@fancyapps/ui` v6 is already installed
- Reusable `useFancybox` hook exists at `src/hooks/useFancybox.ts` (lazy-loads Fancybox via IntersectionObserver, handles cleanup)

- API data mapping for `highlighted_transformations` doesn't extract video URLs

### 3. Integrate Fancybox using the existing `useFancybox` hook
**File:** `merged/src/landings/lifestyle/sections/TransformationVideos.jsx`

Following the exact pattern used in `Hero.jsx` and `VideoTestimonials.tsx`:

- Import `useFancybox` from `@/hooks/useFancybox`
- Define a `FANCYBOX_SELECTOR` constant: `"[data-fancybox='transformation-videos']"`
- Call `useFancybox(FANCYBOX_SELECTOR, { ...options })` with:
  - `Toolbar: false`
  - `animated: true`
  - `dragToClose: true`
  - `closeButton: true`
  - `Youtube: { autoplay: 1 }`
- Attach the returned `ref` to the `<section>` element

### 4. Convert cards from `<button>` to `<a>` with Fancybox data attributes
**File:** `merged/src/landings/lifestyle/sections/TransformationVideos.jsx`

Change each `<SwiperSlide>` card from `<button>` to `<a>`:
```jsx
<a
  href={v.youtubeUrl}
  data-fancybox="transformation-videos"
  aria-label="شغّل فيديو التحول"
  className="relative block w-full aspect-[9/14] rounded-2xl overflow-hidden group shadow-card cursor-pointer"
>
  {/* existing img, gradient overlay, play button */}
</a>
```

This follows the exact pattern in `biomedical/VideoTestimonials.tsx` where `<a>` tags with `data-fancybox` and YouTube `href` are used inside Swiper slides.
