# Plan: Adapt Lifestyle Hero to Use Shared Video Utilities (Same Logic as Biomedical)

## Problem

The lifestyle `Hero.jsx` uses a **hardcoded YouTube video ID** (`5nz32pPBq8w`) with a **custom YouTube IFrame API implementation** for desktop inline playback and **Fancybox** for mobile modal. This is:

1. Not dynamic вАФ video URL cannot be changed from the API
2. YouTube-only вАФ no Vimeo or MP4 support
3. Uses a heavy custom YT IFrame API integration (50+ lines) when shared utilities already exist
4. Uses Fancybox (external dependency) instead of the shared `VideoPlayerModal` component

The biomedical `Hero.tsx` already has the correct architecture using shared video utilities (`videoUtils.ts`, `VideoPlayButton`, `VideoPlayerModal`). The lifestyle Hero should follow the same pattern.

## File to Modify

Only `merged/src/landings/lifestyle/sections/Hero.jsx` needs changes.

No changes to `videoUtils.ts`, `VideoPlayButton.tsx`, `VideoPlayerModal.tsx`, or `useFancybox.ts`.

## Architecture Differences (Current vs Target)

| Aspect | Current Lifestyle | Target (Same as Biomedical) |
|--------|-------------------|-----------------------------|
| Video URL | Hardcoded `YOUTUBE_VIDEO_ID` | `apiData?.general_data?.[0]?.hero_video` from API |
| Desktop inline play | Custom YouTube IFrame API (YT.Player) | `getEmbedUrl()` + `<iframe>` / `<video>` via `videoUtils` |
| Desktop play button | Custom `FaPlay`/`FaPause` with pulse animation | Shared `VideoPlayButton` component |
| Mobile modal | Fancybox (`useFancybox` hook) | Shared `VideoPlayerModal` component |
| Mobile background | YouTube-only `<iframe>` | Platform-aware via `getEmbedUrl(..., "background")` |
| Video platform support | YouTube only | YouTube + Vimeo + MP4 |
| Thumbnail | Manual YouTube thumbnail URL | `useVideoThumbnail()` hook |

### 6. Fallback When No Video

If `hero_video` is not set in the API, `hasVideo` will be `false`:
- **Mobile**: No background video, just the gradient overlay (existing `bg-black/60`)
- **Desktop**: Show the orange content column full-width or show the video column empty

This preserves backward compatibility вАФ if no video is configured, the section still renders correctly.
