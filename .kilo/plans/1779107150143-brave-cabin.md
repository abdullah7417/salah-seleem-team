## Key Architectural Patterns to Document

```
URL: /$slug
  → routes/$slug.tsx (TanStack Router)
    → LandingRenderer.jsx
      → getLandingConfig(slug) from config/landings.js
      → ThemeProvider (sets CSS vars per theme)
      → LandingDataProvider (fetches API data via useLandingApi)
      → Renders sections in order from config
        → Each section lazy-loaded via sections.js SECTION_MAP
      → Optional layout components (Footer, WhatsApp)
```

File structure per landing:
```
src/landings/<landing-key>/
  sections.js       ← SECTION_MAP (key → lazy component)
  sections/         ← individual section components (.jsx/.tsx)
  data/             ← static data (fallbacks when API unavailable)
  components/       ← landing-specific shared components (Footer, etc.)
```

Shared infrastructure:
```
src/config/landings.js    ← landing config registry
src/LandingRenderer.jsx   ← renders any landing from config
src/api/client.js         ← fetch wrapper with Bearer auth
src/hooks/useLandingApi.js ← data fetching hook
src/context/LandingDataContext.jsx ← provides API data to sections
src/providers/ThemeProvider.jsx    ← applies theme CSS variables
src/themes/<key>.js                ← CSS variable definitions
src/components/common/            ← shared components (Container, CTAButton, etc.)
```
