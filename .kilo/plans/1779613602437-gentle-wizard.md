# Plan: Lifestyle Pricing — Membership Plans Selection Flow

## Current State

`merged/src/landings/lifestyle/sections/Pricing.jsx` reads from `apiData.memberships` (flat array) and renders a single orange card with a native `<select>` dropdown to pick a plan. Features come from `apiData.membership_plans[0].features`.

## Goal

Refactor to use the nested `apiData.membership_plans[].memberships[]` structure. Each `membership_plan` becomes a card. Inside each card, memberships are shown as selectable radio-style options. Pay button navigates to `/membership/membership/payment/plans/{planId}/{membershipId}`.

---

## Implementation

### 1. Update data derivation (`Pricing.jsx`)

Replace the flat `pricingPlans` memo with a new `membershipPlans` memo that maps `apiData.membership_plans`:

```js
const membershipPlans = useMemo(() => {
  const plans = apiData?.membership_plans;
  if (!plans?.length) return fallbackPlans; // static fallback
  return plans;
}, [apiData]);
```

Keep `pricingFeatures` memo as-is (already reads from `membership_plans[0].features`). Also add a per-plan features fallback.

### 2. State management

Replace the flat `planIdx` state with per-plan membership selection:

```js
// Map<planId, membershipId> — stores which membership is selected per plan
const [selections, setSelections] = useState({});

// Initialize defaults when membershipPlans changes
useEffect(() => {
  const defaults = {};
  membershipPlans.forEach(plan => {
    const popular = plan.memberships?.find(m => m.most_popular === 1);
    defaults[plan.id] = popular?.id ?? plan.memberships?.[0]?.id ?? null;
  });
  setSelections(prev => ({ ...defaults, ...prev }));
}, [membershipPlans]);
```

Helper to get the full selected membership object:

```js
const getSelectedMembership = (plan) =>
  plan.memberships?.find(m => m.id === selections[plan.id]) ?? null;
```

### 3. Replace `<select>` with styled radio options

Inside each plan card, replace the native `<select>` dropdown with selectable radio-style buttons matching the existing orange/amber theme. Each option shows `membership.title` and has an active state with smooth CSS transition:

- Unselected: `bg-[#fdf6ef]` with subtle border
- Selected: `bg-[#fdf6ef]` with `ring-2 ring-primary` and check indicator
- Transition: `transition-all duration-200`

This keeps the card structure identical but upgrades the selection UX.

### 4. Dynamic data display

When selection changes, update dynamically within each plan card:

- Price (`plan.price` → `selectedMembership.price`)
- Before price
- Duration/months
- Details (if rendered)
- "Most popular" badge (conditional on `selectedMembership.most_popular === 1`)
- Pay button text

All using the full membership object from `getSelectedMembership(plan)`.

### 5. Pay button navigation

Replace the current `<button>` with navigation logic:

```js
const handlePay = (plan) => {
  const membership = getSelectedMembership(plan);
  if (!membership) return;
  window.location.href = `https://salahseleemteam.com/membership/membership/payment/plans/${plan.id}/${membership.id}`;
};
```

Using `window.location.href` since this is an external route (not a TanStack route).

Disable the button if no membership is selected.

### 6. Handle multiple membership_plans

If `membershipPlans` has >1 plan, wrap cards in a grid layout:

- 1 plan: `max-w-2xl mx-auto` (current)
- 2+ plans: `grid grid-cols-1 md:grid-cols-2 gap-6 max-w-4xl mx-auto`

This preserves the single-plan look when only one plan exists and gracefully scales.

### 7. Edge cases

- `memberships` undefined/null → show nothing for that plan card (or skip plan entirely)
- Empty memberships array → skip rendering that plan's card
- Missing prices → show `"—"` or `0`
- Missing titles → show empty string
- API loading/null → fallback to static data
- `selections[plan.id]` is null → disable pay button

### 8. Static fallback

When `apiData.membership_plans` is empty/missing, convert `staticPlans` into the same shape so the existing fallback works:

```js
const fallbackPlans = [{
  id: 0,
  title: "تحدي الــ 90 يوم",
  subtitle: "",
  memberships: staticPlans.map(p => ({
    id: p.id,
    title: p.label,
    months: p.duration,
    price: p.price,
    price_before: p.before,
    most_popular: 0,
  })),
}];
```
