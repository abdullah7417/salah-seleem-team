## Implementation Plan

### 1. Create `useMediaQuery` hook (reusable utility)

**File**: `merged/src/hooks/useMediaQuery.ts`

- Custom hook that accepts a breakpoint value, returns a boolean
- Uses `useState` + `useEffect` with `matchMedia` for proper SSR-safe detection
- Debounces resize events with `requestAnimationFrame` (pattern already used in biomedical Hero)
- Initializes state inside `useEffect` to prevent hydration mismatch
- Replace existing `useIsMobile` usage or extend it — but since the project already has `useIsMobile` at `src/hooks/use-mobile.tsx`, I'll enhance it to support the `lg` (1024px) breakpoint rather than creating a separate hook. Actually, the existing `useIsMobile` already accepts a `breakpoint` param. I'll just call `useIsMobile(1024)` from the Hero.

**Decision**: Use the existing `useIsMobile` hook with `breakpoint=1024`. No new hook needed.

### 2. Create `useFancybox` hook (reusable utility)

**File**: `merged/src/hooks/useFancybox.ts`

A reusable hook that encapsulates the Fancybox lazy-loading pattern already used in biomedical Hero:

```ts
function useFancybox(selector: string, options?: object): { ref, ready }
```

- Uses `IntersectionObserver` to lazy-load Fancybox CSS + JS only when the ref element enters viewport (with 200px rootMargin)
- Dynamically imports `@fancyapps/ui` and `@fancyapps/ui/dist/fancybox/fancybox.css`
- Calls `Fancybox.bind()` with provided options
- Cleans up `Fancybox.unbind()` + `Fancybox.close()` on unmount
- Returns `ready` boolean to allow conditional rendering
- Prevents double-binding via internal flag

This extracts the pattern from `biomedical/sections/Hero.tsx:146-169` into a reusable hook.

#### Desktop Behavior (`lg+`, right image column)

**State**: `desktopVideoState: 'idle' | 'playing' | 'paused'`
- Prevents double-trigger with a `isTransitioning` ref that blocks rapid clicks for 300ms

**Idle state**:
- Show YouTube thumbnail image (instead of `coachHero.jpg`)
- Fallback chain: `maxresdefault.jpg` → `hqdefault.jpg` → `coachHero.jpg` on error
- Show animated play button overlay (existing pulsing design, enhanced with hover scale)
- Dark gradient overlay on top of thumbnail for premium feel

**Playing state**:
- Fade in YouTube iframe with `autoplay=1&rel=0&modestbranding=1`
- Hide play button, show a semi-transparent pause icon overlay on hover
- Iframe positioned absolutely to fill the image column (same layout, no content shift)

**Paused state**:
- YouTube iframe remains visible but paused
- Show centered play button overlay again
- Click → resume playing

**Implementation approach for play/pause**:
- Use the YouTube IFrame Player API loaded dynamically
- `window.YT.Player` instance created after iframe mounts
- `player.playVideo()` / `player.pauseVideo()` for toggle control
- If YT API isn't available (rare), fallback to removing/re-adding iframe src with/without autoplay

**Transition**:
- CSS `opacity` transition on iframe (fade in from poster)
- Play button scales down on click via Framer Motion `animate`
- No layout shift — iframe overlays the thumbnail position absolutely

## Key Design Decisions

1. **YouTube IFrame API for desktop play/pause**: Gives proper toggle control vs. iframe src swap hack
2. **Reuse `useIsMobile(1024)`**: The `lg` breakpoint in Tailwind is 1024px, matching the existing responsive split
3. **Extract `useFancybox` hook**: The same IntersectionObserver + dynamic import pattern is used in biomedical Hero — extracting it avoids duplication and makes the lifestyle Hero cleaner
4. **Poster → iframe transition on desktop**: CSS opacity transition preserves layout (no content shift)
5. **Fancybox only on mobile**: Desktop inline play provides a better UX on large screens; Fancybox is only loaded when the mobile view is active
6. **Thumbnail fallback**: `onError` handler falls back from `maxresdefault.jpg` to `hqdefault.jpg`

## Accessibility Considerations

- `aria-label` on play/pause buttons
- `role="button"` and `tabIndex={0}` on clickable overlays
- Keyboard support (`Enter`/`Space`) for desktop video toggle
- Proper `alt` text on thumbnail images
- `title` attribute on iframe

## Performance Considerations

- Fancybox JS/CSS loaded only when hero section enters viewport (IntersectionObserver with 200px margin)
- YouTube IFrame API loaded only on first desktop play click
- No autoplay on initial load (requirement)
- `passive: true` on resize listener
- `requestAnimationFrame` debounce on resize
- YouTube iframe only mounted when needed (desktop playing state)
- Thumbnail image uses `loading="eager"` since it's above the fold
